# 126 HV2 -- Hyperloop Pod Shell

Submission-agnostic CalculiX eval kit for a 7075-T73 aluminum cylindrical
pod shell sized for a collegiate low-pressure transport pod (host tube
2.5 m ID at 100 Pa absolute, pod interior at sea-level pressure).

## Item under test

A monolithic, thin-walled cylindrical pod shell.  The reference geometry in
`build.py` is a uniform 5 mm wall hollow cylinder; the spec also describes
integral T-stringers, ring frames, and bolted end flanges, but the
submission-agnostic deck only requires that the agent's STEP expose:

| selector | role                              |
|----------|-----------------------------------|
| `NEND1`  | z_min end face -- clamped 1-3     |
| `NEND2`  | z_max end face -- clamped 1-3     |
| `NINNER` | inner cylindrical surface (DSLOAD) |

| parameter        | value         |
|------------------|---------------|
| outer diameter   | 1800 mm       |
| wall thickness   | 5 mm          |
| length           | 2000 mm       |
| material         | Al 7075-T73 (E = 72 GPa, nu = 0.33, rho = 2810 kg/m^3, yield = 435 MPa) |

## Files in this kit

| file                    | purpose |
|-------------------------|---------|
| `spec.json`             | Original eval prompt, requirements, pass/fail thresholds. |
| `build.py`              | cadquery script: builds the hollow shell, exports `out.step` + `meta.json`. |
| `analysis_template.inp` | CalculiX deck (material, BCs by NSET name, 5 steps incl. *BUCKLE + *FREQUENCY). |
| `check.py`              | Post-processor: parses `model.dat`, prints R1-R6 PASS/FAIL/SKIP. |
| `grade_ccx.py`          | Kit-local wrapper around the shared end-to-end runner. |
| `notes.md`              | Modeling notes from the original (per-element, custom-mesh) deck. |
| `model.*`, `*.log`      | Run artifacts (regenerated by `grade_ccx.py`). |

The shared runner lives at `scripts/ccx_eval/grade_ccx.py`.  It runs
`build.py -> gmsh -> wire_bcs.py -> ccx -> check.py`.  The kit-local
`grade_ccx.py` just delegates to it with this directory as workdir.

## Boundary conditions and loading

The submission-agnostic deck does NOT hardcode any node IDs.  `wire_bcs.py`
consumes `meta.json` to emit:

* `*NSET, NSET=NEND1` and `NEND2` -- nodes on the two axial extremes
  (z_min, z_max).  Clamped 1-3 in the deck (idealised bolted flange
  fixity).
* `*NSET, NSET=NINNER` -- nodes on the inner cylindrical surface, picked
  by the `radius_xy` selector at radius r_inner = 895 mm with a 2 mm
  tolerance (excludes the outer cylinder at r_outer = 900 mm with 3 mm
  margin).  Provided for debugging / *NODE PRINT use.
* `*SURFACE, NAME=SINNER, TYPE=ELEMENT` -- every tet face whose three
  corner nodes lie on the inner cylinder, auto-built by `wire_bcs.py`
  from the `pressure_surfaces` block in `meta.json`.  This is what the
  template loads with `*DLOAD SINNER, P, ...`.

Pressure sign convention on `SINNER` (TYPE=ELEMENT, outward normal points
radially inward from the wall element into the cavity):

* `*DLOAD SINNER, P, +0.2`  pushes the wall radially OUTWARD ->
  internal pressurization (LC1).
* `*DLOAD SINNER, P, -0.2`  pushes the wall radially INWARD ->
  membrane-equivalent to external crush on the wall for the open-ended,
  thin (5/900 = 0.55 % wall ratio) shell used here (LC2 + *BUCKLE).

The eval-kit task brief calls these `*DSLOAD`; CCX 2.22 reserves
`*DSLOAD` for shell-edge loads, so on solid C3D elements with a
*SURFACE TYPE=ELEMENT block the equivalent surface-pressure card is
`*DLOAD <surface>, P, magnitude` (sign convention identical).

LC3 cornering uses `*DLOAD GRAV` body force on `Eall` (no NSET selectors
needed): 0.4 g lateral (+x) and 2.0 g vertical (-y), per the cornering
load case in `spec.json`.

## Five-step deck layout

| step | type        | load                                      | criterion |
|------|-------------|-------------------------------------------|-----------|
| 1    | `*STATIC`   | LC1 internal pressurization +0.2 MPa      | R1, R6    |
| 2    | `*STATIC`   | LC2 external crush -0.2 MPa               | R3        |
| 3    | `*BUCKLE`   | LC2 same load, 5 eigenvalues              | R2        |
| 4    | `*STATIC`   | LC3 cornering (0.4 g lateral + 2 g vert)  | R1 (LC3)  |
| 5    | `*FREQUENCY`| free-of-load modal, 5 modes               | R4        |

Step 3 (`*BUCKLE`) does not advance total time, so the static stress
blocks land at total times 1.0, 2.0, 3.0 (LC1, LC2, LC3 respectively);
`check.py` keys off those times.

## Pass/Fail criteria

| id  | metric                                          | limit            | source |
|-----|-------------------------------------------------|------------------|--------|
| R1  | max von Mises stress, LC1                       | <= 218 MPa       | yield/2.0 per benchmark R&R v1.0 SF2.  |
| R1  | max von Mises stress, LC3 cornering             | <= 218 MPa       | same as R1.                            |
| R2  | first buckling load factor, LC2 external crush  | >= 2.0           | R&R v1.0 mandatory SF2 on calculated load. |
| R3  | max von Mises stress, LC2 external crush        | <= 218 MPa       | yield/2.0.                              |
| R4  | first 5 natural frequencies                     | outside 10-40 Hz | track-joint excitation band (3.5 m concrete panels at 100 km/h). |
| R5  | monolithic shell mass                           | <= 180 kg        | pod mass budget.                        |
| R6  | unfactored LC1 stress range, 1000 cycles        | <= 140 MPa       | 7075-T73 endurance limit at 5e8 cycles per MIL-HDBK-5J. |

R5 mass is recomputed analytically from the geometry in `build.py`
(uniform 5 mm wall hollow cylinder, OD 1800 mm x L 2000 mm) at
rho = 2810 kg/m^3, so it is independent of the meshing/discretisation
quality.  Stiffened-shell submissions can override the mass calculation
in `check.py` if their geometry deviates.

## Reproducing

End-to-end (kit-local wrapper):
```
python ./docs/eval/ccx_50/single/126_hv2_hyperloop_pod_shell/grade_ccx.py
```

End-to-end (shared runner):
```
python ./scripts/ccx_eval/grade_ccx.py \
       ./docs/eval/ccx_50/single/126_hv2_hyperloop_pod_shell
```

Re-grade only (results already exist):
```
python ./docs/eval/ccx_50/single/126_hv2_hyperloop_pod_shell/check.py
```

Expected `check.log` summary on the reference uniform-wall geometry
(measured run, mm-MPa-N-tonne unit set, gmsh tet mesh ~10 k nodes):

| id      | metric                          | result          | status |
|---------|---------------------------------|-----------------|--------|
| R1      | LC1 max von Mises               | 52.6 MPa        | PASS   |
| R1-LC3  | LC3 max von Mises               | 0.19 MPa        | PASS   |
| R2      | LC2 first buckling load factor  | 44.8            | PASS   |
| R3      | LC2 max von Mises               | 52.6 MPa        | PASS   |
| R4      | first 5 freqs (Hz)              | 375 ... 515 Hz  | PASS   |
| R5      | monolithic shell mass           | 158.9 kg        | PASS   |
| R6      | unfactored LC1 stress range     | 26.3 MPa        | PASS   |

Overall: PASS.  The buckling load factor of 44.8 reflects the strong
short-cylinder (L/r = 2.2) clamped-clamped boundary condition; an
unstiffened tube of these proportions is far from buckling-critical
once both ends are bolted to rigid flanges.

# 090 NG5 -- ISO 10328 P5 Transtibial Prosthetic Pylon

Submission-agnostic CalculiX eval kit for a below-knee (transtibial)
prosthetic pylon qualified to ISO 10328:2016 test level P5
(high-activity adult, body mass up to 100 kg).

## Item under test

A 6061-T6 aluminum tubular pylon clamped between a proximal pyramid
adapter and a distal pyramid clamp. Only the structurally critical
tube is meshed; the two pyramid adapters are accounted analytically
in the R5 mass check.

| parameter         | value     | spec envelope |
|-------------------|-----------|---------------|
| outer diameter    | 30.0 mm   | 28-34 mm      |
| wall thickness    |  2.5 mm   | 1.5-3.0 mm    |
| length (clamped)  | 250.0 mm  | 200-400 mm    |
| material          | Al 6061-T6 (E=68.9 GPa, nu=0.33, yield=276 MPa, rho=2.7 g/cc) |

## Files in this kit

| file                    | purpose |
|-------------------------|---------|
| `spec.json`             | Original eval prompt, requirements, pass/fail thresholds. |
| `build.py`              | cadquery script: builds the tube, exports `out.step` + `meta.json`. |
| `analysis_template.inp` | CalculiX deck (material, BCs by NSET name, two `*STATIC` steps). |
| `check.py`              | Post-processor: parses `model.frd` and `model.dat`, prints R1-R5 PASS/FAIL/SKIP. |
| `notes.md`              | Historical modeling notes from the original (per-node distributed-force) deck. |
| `model.*`, `*.log`      | Latest run artifacts (regenerated by `grade_ccx.py`). |

The shared runner is `scripts/ccx_eval/grade_ccx.py`. It runs
`build.py` -> gmsh -> `wire_bcs.py` -> ccx -> `check.py`.

## Boundary conditions and loading

The submission-agnostic deck does NOT hardcode any node IDs. Two NSETs
are populated by `wire_bcs.py` from the `meta.json` selectors:

* `NFIXED` -- nodes on the distal face (`z_min`); pinned in DOF 1-3.
  Represents the fully clamped foot-side adapter in the ISO 10328 jig.
* `NLOAD`  -- nodes on the proximal face (`z_max`). A single off-mesh
  reference node (id 999999) is rigidly coupled to NLOAD via
  `*RIGID BODY, NSET=NLOAD, REF NODE=999999`. The principal static
  test force (3360 N TC I, 4000 N TC II) is applied at the reference
  node along `-Z`, so the total load on the structure equals the
  ISO 10328 principal force independent of mesh density.

The Annex A offset bending arms are NOT applied as moments here:
ISO 10328's "principal static test force" (Table 8) is the force
magnitude that must be transmitted; the rig's offset is fixture-side
geometry. R1 stress is consequently dominated by axial compression
(yielding strength margin orders of magnitude); the reaction-force
checks (R2, R3) directly verify that 3360 N / 4000 N is being driven
through the part.

## Pass/Fail criteria

| id | metric                                       | limit            | source |
|----|----------------------------------------------|------------------|--------|
| R1 | max von Mises stress, LC1 and LC2            | <= 276 MPa       | Al 6061-T6 yield, ISO 10328 ultimate-static survival. |
| R2 | reaction Fz under TC I  (toe)  load step     | >= 3360 N        | ISO 10328 P5 Table 8 principal static, TC I. |
| R3 | reaction Fz under TC II (heel) load step     | >= 4000 N        | ISO 10328 P5 Table 8 principal static, TC II. |
| R4 | fatigue life at principal cyclic forces      | >= 3.0e6 cycles  | **SKIP** -- ISO 10328 P5 mandates 3 Mcycle S-N fatigue (LC3 1330 N, LC4 1265 N). The spec's `verification.requires_non_fea_solver` block names fe-safe / nCode / MSC Fatigue; CalculiX 2.22 has no built-in S-N evaluator, so this criterion is intentionally skipped per spec. |
| R5 | assembly mass (pylon + adapters)             | <= 400 g         | Clinical lightweight target; pylon volume from geometry, adapters added analytically (60 g pyramid + 90 g clamp). |

Excluded analyses (`spec.json -> verification.explicitly_excluded`):
buckling, vibration / modal, thermal, fluid, radiation. The pylon
section is well below the slender-column buckling threshold for an
Al tube of this length / diameter.

## Reproducing

```
python ./scripts/ccx_eval/grade_ccx.py \
       ./docs/eval/ccx_50/single/090_ng5_iso10328_p5_prosthetic_pylon
```

Expected `check.log` summary: R1 PASS, R2 PASS, R3 PASS, R4 SKIP,
R5 PASS, OVERALL PASS.
